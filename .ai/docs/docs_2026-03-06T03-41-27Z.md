5. **Config Format**: Arrays for enabled/disabled skills in OpenClaw config
6. **Security**: Admin-only access with audit logging

## API Documentation Summary

### Endpoints Documented

1. **GET `/api/skills`**
   - Lists all discovered skills with states
   - Requires admin role
   - Returns array of Skill objects

2. **PUT `/api/skills`**
   - Updates skill enabled/disabled states
   - Requires admin role
   - Accepts batch updates
   - Logs audit events

### Request/Response Examples

#### GET Request
```bash
curl -H "Cookie: mc-session=token" http://localhost:3000/api/skills
```

Response:
```json
{
  "skills": [
    {
      "name": "frontend-design",
      "description": "Create distinctive frontend interfaces",
      "enabled": true,
      "path": "/home/user/.claude/skills/frontend-design"
    }
  ]
}
```

#### PUT Request
```bash
curl -X PUT http://localhost:3000/api/skills \
  -H "Content-Type: application/json" \
  -H "Cookie: mc-session=token" \
  -d '{"updates": {"frontend-design": true}}'
```

Response:
```json
{
  "ok": true,
  "count": 1
}
```

### Error Codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 400 | Invalid request body |
| 401 | Authentication required |
| 403 | Admin access required |
| 429 | Rate limit exceeded |
| 500 | Server error |

### Authentication Methods

1. Session cookie (`mc-session`)
2. API key header (`x-api-key`)

Both require admin role for skills endpoints.
