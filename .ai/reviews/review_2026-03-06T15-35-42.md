### ⚠️ HIGH RISK (Should Fix Before Merge)

#### Issue #1: Variable Shadowing in getAllSkills()

**Description**: The function `getAllSkills()` declares `const config = getSkillsConfig()` which shadows the imported `config` from `'./config'` at the top of the file. While this doesn't cause a bug in the current implementation (since the function only uses the local variable), it creates confusion and could lead to bugs if future changes need to access `config.claudeHome` or other imported config properties within this function.

**Location**:
- File: `src/lib/skills.ts`
- Lines: `179`

**Offending Code**:
```typescript
import { config } from './config'  // Line 3
// ...
export function getAllSkills(): Skill[] {
  const discoveredSkills = discoverSkills()
  const config = getSkillsConfig()  // Line 179 - shadows import
```

**Recommended Solutions**:
1. **Rename local variable** (Preferred)
   - Change `const config = getSkillsConfig()` to `const skillsConfig = getSkillsConfig()`
   - Update references: `skillsConfig.enabled`, `skillsConfig.disabled`
   - Rationale: Maintains clarity about which config is being used

---

#### Issue #2: Toggle Buttons Lack Accessibility Attributes

**Description**: The toggle switch buttons in the SkillsPanel component are missing proper ARIA attributes for screen reader accessibility. The buttons have `title` attributes but lack `role="switch"` and `aria-checked` which are essential for assistive technologies to understand the toggle state.

**Location**:
- File: `src/components/panels/skills-panel.tsx`
- Lines: `215-227`

**Offending Code**:
```tsx
<button
  onClick={() => handleToggle(skill.name, currentEnabled)}
  className={`w-10 h-5 rounded-full relative transition-colors ${
    currentEnabled ? 'bg-violet-500' : 'bg-muted'
  }`}
  title={currentEnabled ? 'Enabled' : 'Disabled'}
>
```

**Recommended Solutions**:
1. **Add ARIA attributes** (Preferred)
   - Add `role="switch"` and `aria-checked={currentEnabled}` to the button
   - Add `aria-label={`Toggle ${skill.name}`}` for better screen reader context
   ```tsx
   <button
     onClick={() => handleToggle(skill.name, currentEnabled)}
     role="switch"
     aria-checked={currentEnabled}
     aria-label={`Toggle ${skill.name}`}
     className={...}
   >
   ```
   - Rationale: Follows WAI-ARIA best practices for toggle switches
